Algorithm explanation

The main strategy is decoupling the segments that have to be copied from the actual copy operation. So there is a function in charge of generating all the segments that will be copied and another function that does the actual copying. The copy operation can be made in parallel using WebWorkers (similar to Threads), to improve the performance in very large arrays, the workers don't send large chunks of data back to the main thread, instead they use a SharedBuffer when all thread can write at the same time to the Same Shared Int16Array. There is no risk of collisions since each worker will write to different parts of the array, and each segment doesn't depend on other segments.
